## Using Custom Networks

To use a network not in the supported list:

1. **Deploy the module** on the target network
2. **Use `--rpc-url`** to specify custom RPC endpoint
3. **Specify `--token-list-strategy chain`** (explorer won't work)
4. **Verify CoW Protocol** is deployed on that network

**Example:**
```bash
yarn ts-node index.ts \
  --network mainnet \
  --rpc-url https://custom-network-rpc.com \
  --module 0xYourModuleAddress \
  --token-list-strategy chain \
  --lookback-range 1000
```

<Warning>
Ensure:
- CoW Protocol Settlement contract exists on the network
- Your Safe is whitelisted as a solver
- The network supports EIP-1559 transactions (or modify gas configuration)
</Warning>

## Verifying Deployments

Verify a module deployment:

```bash
NETWORK=mainnet  # or gnosis, arbitrum, etc.
MODULE_ADDRESS=0x...

# Get RPC from config or use custom
RPC_URL=https://eth.llamarpc.com

# Verify contract exists
cast code $MODULE_ADDRESS --rpc-url $RPC_URL

# Check module parameters
cast call $MODULE_ADDRESS "settlement()(address)" --rpc-url $RPC_URL
cast call $MODULE_ADDRESS "keeper()(address)" --rpc-url $RPC_URL
cast call $MODULE_ADDRESS "receiver()(address)" --rpc-url $RPC_URL
cast call $MODULE_ADDRESS "minOut()(uint256)" --rpc-url $RPC_URL
```

## Network-Specific Considerations

<Accordion title="Mainnet">
  **Gas Costs:** Highest among all networks
  
  **Recommendations:**
  - Use explorer strategy for faster execution
  - Set higher `minOut` to avoid unprofitable swaps
  - Consider gas price optimization
  - Use paid RPC provider for reliability
</Accordion>

<Accordion title="Gnosis Chain">
  **Gas Token:** xDAI (stable, low fees)
  
  **Recommendations:**
  - Use explorer strategy (Blockscout)
  - Lower `minOut` acceptable due to cheap gas
  - Fast block times (~5s)
  - Good for frequent keeper runs
</Accordion>

<Accordion title="Arbitrum">
  **L2 Network:** Very low gas costs
  
  **Recommendations:**
  - Use chain strategy (no explorer API)
  - Higher lookback range due to fast blocks (~0.25s)
  - Very low `minOut` feasible
  - Frequent keeper runs economical
</Accordion>

<Accordion title="Base">
  **L2 Network:** Low gas costs, fast blocks
  
  **Recommendations:**
  - Use chain strategy
  - Moderate lookback range
  - Low `minOut` acceptable
  - Growing ecosystem
</Accordion>

<Accordion title="Sepolia (Testnet)">
  **Testing Network:** Free testnet tokens
  
  **Recommendations:**
  - Perfect for testing deployments
  - Use chain strategy
  - Get testnet ETH from faucets
  - Test module enablement flow
</Accordion>

## Adding New Networks

To add support for a new network:

1. **Update `ts/config.ts`:**
   ```typescript
   export const SUPPORTED_NETWORKS = [
     ...,
     "newnetwork",
   ] as const;
   
   export const networkSpecificConfigs = {
     ...,
     newnetwork: {
       rpcUrl: "https://rpc.newnetwork.com",
       explorer: "https://explorer.cow.fi/newnetwork",
       chainId: SupportedChainId.NEWNETWORK,
     },
   };
   ```

2. **Deploy module** on the new network

3. **Update `networks.json`:**
   ```json
   {
     "COWFeeModule": {
       ...,
       "<chain-id>": {
         "address": "0x...",
         "transactionHash": "0x..."
       }
     }
   }
   ```

4. **Test the deployment** on testnet first
