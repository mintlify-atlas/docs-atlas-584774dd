This guide walks through deploying the COWFeeModule contract using Foundry's forge script.

## Prerequisites

Before deploying, ensure:

- [x] [Prerequisites](/setup/prerequisites) are met
- [x] [Environment variables](/setup/environment) are configured
- [x] `.env` file is populated with correct values
- [x] Deployer wallet has sufficient native tokens for gas

## Deployment Process

<Steps>
  <Step title="Load Environment Variables">
    Source your `.env` file to load configuration:
    
    ```bash
    source .env
    ```
  </Step>
  
  <Step title="Dry Run Deployment">
    Test the deployment without broadcasting to the network
  </Step>
  
  <Step title="Deploy Contract">
    Broadcast the deployment transaction to the network
  </Step>
  
  <Step title="Verify Contract (Optional)">
    Verify the contract source code on block explorers
  </Step>
</Steps>

## Dry Run Deployment

First, simulate the deployment to verify parameters and estimate gas:

**From `README.md:31-36`:**
```bash
forge script ./script/DeployCOWFeeModule.s.sol \
  --rpc-url <rpc>
```

**Example:**
```bash
forge script ./script/DeployCOWFeeModule.s.sol \
  --rpc-url $RPC_URL
```

<Note>
The dry run simulates the deployment locally without sending any transactions. This helps verify:
- Constructor parameters are correct
- Contract compiles successfully
- Gas estimation
- No deployment errors
</Note>

**Expected output:**
```
[⠒] Compiling...
[⠢] Compiling 1 files with 0.8.26
[⠆] Solc 0.8.26 finished in 1.23s
Compiler run successful!

Script ran successfully.
Gas used: 1234567

== Return ==
0: address 0x...
```

## Deploy and Verify

Once the dry run succeeds, deploy the contract to the network:

**From `README.md:38-42`:**
```bash
forge script ./script/DeployCOWFeeModule.s.sol \
  --rpc-url <rpc> \
  --broadcast --verify --private-key <private-key>
```

<Warning>
Make sure to set the `ETHERSCAN_API_KEY` environment variable for verification to work.
</Warning>

**Example:**
```bash
forge script ./script/DeployCOWFeeModule.s.sol \
  --rpc-url $RPC_URL \
  --broadcast \
  --verify \
  --private-key $DEPLOYER_PRIVATE_KEY
```

<Accordion title="Alternative: Use --account instead of --private-key">
Foundry supports hardware wallets and encrypted keystores:

```bash
# Using a named account (more secure)
forge script ./script/DeployCOWFeeModule.s.sol \
  --rpc-url $RPC_URL \
  --broadcast \
  --verify \
  --account myaccount
```

See [Foundry Book - Account Management](https://book.getfoundry.sh/reference/cast/cast-wallet) for details.
</Accordion>

## Manual Verification

If automatic verification fails, verify manually:

**From `README.md:46-52`:**
```bash
source .env

forge verify-contract <fee-module-address> \
  COWFeeModule \
  --chain-id <chain-id> \
  --etherscan-api-key $ETHERSCAN_API_KEY \
  --constructor-args $(cast abi-encode "constructor(address,address,address,address,bytes32,address,uint256)" $SETTLEMENT $TARGET_SAFE $WRAPPED_NATIVE_TOKEN $KEEPER $APP_DATA $RECEIVER $MIN_OUT)
```

**Example for Mainnet (Chain ID 1):**
```bash
source .env

forge verify-contract 0x1234567890abcdef1234567890abcdef12345678 \
  COWFeeModule \
  --chain-id 1 \
  --etherscan-api-key $ETHERSCAN_API_KEY \
  --constructor-args $(cast abi-encode \
    "constructor(address,address,address,address,bytes32,address,uint256)" \
    $SETTLEMENT \
    $TARGET_SAFE \
    $WRAPPED_NATIVE_TOKEN \
    $KEEPER \
    $APP_DATA \
    $RECEIVER \
    $MIN_OUT)
```

**Constructor Arguments (from `src/COWFeeModule.sol:43-51`):**

| Position | Parameter | Type | Description |
|----------|-----------|------|-------------|
| 1 | `_settlement` | `address` | CoW Protocol Settlement contract |
| 2 | `_targetSafe` | `address` | Gnosis Safe address |
| 3 | `_wrappedNativeToken` | `address` | Wrapped native token (WETH, etc.) |
| 4 | `_keeper` | `address` | Keeper wallet address |
| 5 | `_appData` | `bytes32` | CoW order metadata hash |
| 6 | `_receiver` | `address` | Fee recipient address |
| 7 | `_minOut` | `uint256` | Minimum output threshold |

## Validate Deployment Parameters

If updating an existing deployment, compare parameters with the previous version:

**From `README.md:56-63`:**
```bash
yarn ts-node compare-deployments.ts \
  --network <network name> \
  <address of the newly deployed module>
```

**Example:**
```bash
yarn ts-node compare-deployments.ts \
  --network mainnet \
  0x1234567890abcdef1234567890abcdef12345678
```

This script compares the new deployment with the address in `networks.json` and displays any parameter changes.

## Verify Deployment

After deployment, verify the contract state:

```bash
# Set the deployed module address
MODULE_ADDRESS=0x1234567890abcdef1234567890abcdef12345678

# Verify constructor parameters
cast call $MODULE_ADDRESS "settlement()(address)" --rpc-url $RPC_URL
cast call $MODULE_ADDRESS "targetSafe()(address)" --rpc-url $RPC_URL
cast call $MODULE_ADDRESS "wrappedNativeToken()(address)" --rpc-url $RPC_URL
cast call $MODULE_ADDRESS "keeper()(address)" --rpc-url $RPC_URL
cast call $MODULE_ADDRESS "appData()(bytes32)" --rpc-url $RPC_URL
cast call $MODULE_ADDRESS "receiver()(address)" --rpc-url $RPC_URL
cast call $MODULE_ADDRESS "minOut()(uint256)" --rpc-url $RPC_URL
```

**Verify each matches your `.env` configuration.**

<Note>
All parameters are immutable after deployment. If you need to change any parameter, you must deploy a new module instance.
</Note>

## Troubleshooting

<Accordion title="Error: Insufficient funds for gas">
  Ensure the deployer wallet has enough native tokens:
  
  ```bash
  cast balance <deployer-address> --rpc-url $RPC_URL
  ```
  
  Transfer more native tokens to the deployer address.
</Accordion>

<Accordion title="Error: Verification failed">
  Manual verification options:
  
  1. Use the `forge verify-contract` command (see Manual Verification section)
  2. Verify via block explorer UI by uploading source code
  3. Check that `ETHERSCAN_API_KEY` is set correctly
  4. Ensure you're using the correct explorer (Etherscan, Gnosisscan, etc.)
</Accordion>

<Accordion title="Error: Contract creation failed">
  Common causes:
  
  - Invalid constructor parameters (check `.env` values)
  - Insufficient gas limit
  - Network congestion
  - RPC endpoint issues
  
  Review the error message and verify all parameters.
</Accordion>
