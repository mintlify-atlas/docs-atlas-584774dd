## Overview

This guide covers deploying the COWFeeModule contract using Foundry, verifying it on block explorers, and enabling it on a Safe.

## Deployment Script

The module uses a Foundry script for deployment:

```solidity script/DeployCOWFeeModule.s.sol
contract DeployCOWFeeModule is Script {
    function run() external {
        address receiver = vm.envAddress("RECEIVER");
        address wrappedNativeToken = vm.envAddress("WRAPPED_NATIVE_TOKEN");
        address keeper = vm.envAddress("KEEPER");
        address settlement = vm.envAddress("SETTLEMENT");
        bytes32 appData = vm.envBytes32("APP_DATA");
        bool shouldEnableModule = vm.envOr("SHOULD_ENABLE_MODULE", false);
        address targetSafe = vm.envAddress("TARGET_SAFE");
        uint256 minOut = vm.envUint("MIN_OUT");

        vm.broadcast();
        COWFeeModule module =
            new COWFeeModule(settlement, targetSafe, wrappedNativeToken, keeper, appData, receiver, minOut);

        // only useful for local anvil setup
        if (shouldEnableModule) {
            vm.broadcast(receiver);
            ISafe(receiver).enableModule(address(module));
        }
    }
}
```

## Environment Variables

Create a `.env` file with the following variables:

<ParamField path="RECEIVER" type="address">
  Destination address that will receive the wrapped native tokens.
</ParamField>

<ParamField path="WRAPPED_NATIVE_TOKEN" type="address">
  Address of the wrapped native token:
  - Ethereum: WETH `0xC02aaA39b223FE8D0A0e5C4F27eAD9083C756Cc2`
  - Gnosis Chain: WXDAI `0xe91D153E0b41518A2Ce8Dd3D7944Fa863463a97d`
  - Arbitrum: WETH `0x82aF49447D8a07e3bd95BD0d56f35241523fBab1`
  - Base: WETH `0x4200000000000000000000000000000000000006`
</ParamField>

<ParamField path="KEEPER" type="address">
  Address authorized to execute module operations. This should be the address of the keeper bot.
</ParamField>

<ParamField path="SETTLEMENT" type="address">
  CoW Protocol Settlement contract address:
  - Ethereum: `0x9008D19f58AAbD9eD0D60971565AA8510560ab41`
  - Gnosis Chain: `0x9008D19f58AAbD9eD0D60971565AA8510560ab41`
  - Arbitrum: `0x9008D19f58AAbD9eD0D60971565AA8510560ab41`
  - Base: `0x9008D19f58AAbD9eD0D60971565AA8510560ab41`
</ParamField>

<ParamField path="APP_DATA" type="bytes32">
  CoW Protocol app data identifier for orders created by this module.
</ParamField>

<ParamField path="TARGET_SAFE" type="address">
  The Safe address that will use this module.
</ParamField>

<ParamField path="MIN_OUT" type="uint256">
  Minimum output amount in wei. Orders with buyAmount below this will revert.
</ParamField>

<ParamField path="SHOULD_ENABLE_MODULE" type="bool" default="false">
  Whether to automatically enable the module (only for local testing).
</ParamField>

## Deployment Commands

### Dry Run

Test the deployment without broadcasting transactions:

```bash
forge script ./script/DeployCOWFeeModule.s.sol \
  --rpc-url <rpc>
```

### Deploy and Verify

Deploy the contract and verify on block explorer:

```bash
# Set your API key
export ETHERSCAN_API_KEY=<your-api-key>

forge script ./script/DeployCOWFeeModule.s.sol \
  --rpc-url <rpc> \
  --broadcast --verify --private-key <private-key>
```

<Warning>
Never commit private keys to version control. Use environment variables or hardware wallets.
</Warning>

## Manual Verification

If the contract is already deployed and you need to verify it:

```bash
source .env

forge verify-contract <fee-module-address> COWFeeModule \
  --chain-id <chain-id> \
  --etherscan-api-key $ETHERSCAN_API_KEY \
  --constructor-args $(cast abi-encode "constructor(address,address,address,address,bytes32,address,uint256)" \
    $SETTLEMENT $TARGET_SAFE $WRAPPED_NATIVE_TOKEN $KEEPER $APP_DATA $RECEIVER $MIN_OUT)
```

<Note>
Constructor arguments must be ABI-encoded in the exact order they appear in the constructor.
</Note>

## Enabling the Module

After deployment, the module must be enabled on the target Safe.

### Using Safe Transaction Builder

1. Go to the [Safe Transaction Builder](https://app.safe.global/share/safe-app?appUrl=https%3A%2F%2Fapps-portal.safe.global%2Ftx-builder)
2. Select your target Safe
3. Click "Use Implementation ABI"
4. Select `enableModule` from the dropdown
5. Enter the COWFeeModule address
6. Click "Add new transaction"
7. Send batch and confirm with required signers

### Using Cast

Generate the calldata:

```bash
cast calldata "enableModule(address)" <module-address>
```

Use this calldata to create a transaction from the Safe to itself.

<Warning>
The module must be enabled before the keeper can execute any operations. Verify the module is enabled using Safe's UI or by calling `getModules()` on the Safe contract.
</Warning>

## Validating Deployment Parameters

To compare parameters of a new deployment with an existing one:

```bash
yarn ts-node compare-deployments.ts --network <network-name> <new-module-address>
```

This is useful when redeploying the module with updated parameters.
