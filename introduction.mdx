## What is COW Fee Module?

COW Fee Module is an infrastructure solution that automates fee collection for CoW Protocol. It combines a Solidity smart contract module with a CLI keeper tool to continuously convert accumulated protocol fees into wrapped native tokens across multiple chains.

The module integrates with Gnosis Safe to enable trustless, permission-controlled automation of token swaps through the CoW Protocol settlement contract.

## How It Works

The COW Fee Module operates in three main stages:

1. **Token Discovery** — The keeper identifies tokens held by the CoW Protocol settlement contract using either the Ethplorer/Blockscout API or on-chain Trade events

2. **Filtering & Quoting** — Tokens are filtered based on minimum output thresholds, quoted through the CoW Protocol API, and validated against slippage parameters

3. **Execution** — The keeper submits orders to the CoW OrderBook API and executes a transaction calling `drip()` to approve tokens and commit presignatures for the swap orders

<Note>
All operations are secured by keeper-only permissions enforced at the smart contract level. The module executes transactions through the Safe's `execTransactionFromModule` interface.
</Note>

## Architecture Overview

The system consists of two main components:

- **COWFeeModule.sol** — Solidity smart contract that handles token approvals, order presignatures, and native token wrapping through Safe module execution

- **Keeper CLI** — TypeScript-based automation tool that monitors settlement contract balances, posts orders to CoW Protocol, and triggers the drip operation

The module is deployed with immutable parameters including the target Safe address, wrapped native token, keeper address, minimum output threshold, and app data hash.
